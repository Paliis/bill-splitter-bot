- **Майстер витрати в групі**: `/cancel` і скидання FSM при `/menu`, `/start`, `/help`; інтерпретація суми з першого токена (`100 грн`); якщо в кроці «сума» надходить «розмовний» текст — автоматичне закриття майста замість повторюваної помилки.
- Головне меню: кнопка **«Вийти з майстра»** / **Exit wizard** — те саме скидання FSM, що й `/cancel`.
- Крок суми витрати: розпізнавання **«Кава 100»** (сума останнім словом) і **«100 кава»** (опис після числа) — одразу перехід до вибору учасників з описом; «розмовна» евристика не спрацьовує, якщо останнє слово — число.
- **Двомовність (uk/en)**: `language_code` з Telegram — `uk`/`ru` → українська, інакше англійська; `LocaleMiddleware`. Тексти меню, допомоги, онбордингу, витрат, статусу, завершення — у `services/i18n.py`.
- **Валюта поїздки**: поле `trips.currency` (міграція при старті); для **en** перед назвою — вибір валюти (UAH, USD, EUR, PLN, GBP); **`/new_trip`** без кнопок: uk → UAH, en → USD за замовчуванням. Суми в статусі/витратах у валюті поїздки.
- **Підтримка за мовою**: uk — пріоритет **Mono**; en — **Buy Me a Coffee** (`SUPPORT_BUYMEACOFFEE_URL`), інакше Mono якщо задано.
- **Neon / asyncpg**: з URI прибирається `sslmode` (і `channel_binding`), для SSL передається `connect_args={"ssl": True}` — інакше `connect() got an unexpected keyword argument 'sslmode'`.
- Старт з **SQLite**: попередження в логах про втрату даних після деплою на Render; README/PRODUCT/.env.example — наголос на **PostgreSQL** для продакшену.
- **`/help`**: новий дружній текст (без Group Privacy/адмінів у основному описі); опційна кнопка **«Написати розробнику»** через **`SUPPORT_FEEDBACK_URL`**.
- Меню та онбординг: тепліший текст (користь, кроки 1–3), без переліку команд у повідомленні; прибрано кнопку «Меню» з головної клавіатури; **«Завершити»** коротший підпис; у допомозі — **«До вибору дій»** замість «До меню».
- `/help`: блок «Підтримка» (різний текст з/без посилань); кнопка CTA «Підтримати в Mono».
- Виправлено імпорт **`ExpRefreshMembers`** у `expense_entry` (краш на старті без нього).
- Витрата: кнопка **«Оновити список»** після `/here` або повідомлення; **`Підтвердити`** одразу викликає `callback.answer()`, безпечний HTML у тексті після збереження, fallback `answer` якщо `edit_text` не вдався (щоб не «зависала» кнопка).
- Текст `/help`: уточнено учасників (повідомлення, кнопки, Group Privacy + `/here`); підказка щодо кнопки Mono, якщо не задано `SUPPORT_MONO_URL`.
- Документація релізу: **`SUPPORT_MONO_URL`** у Render Environment; README/PRODUCT для Mono.
- **Middleware учасників**: окремий `TrackGroupMembersMiddleware` (на базі `BaseMiddleware`) робить upsert у `ChatMember` з `message` / `edited_message` / `callback_query` у групі — менше залежності від ручного `/here` (якщо оновлення взагалі доходить до бота; **Group Privacy** це не скасовує).
- **Завершення події**: перед розрахунком переказів нетто по учасниках нормалізуються до суми **0.00** з корекцією копійкової похибки (на найбільшого кредитора або дебітора).
- **Онбординг**: вітання після додавання бота відкладається на **~3.5 с**, щоб кілька поспіль апдейтів (наприклад, багато `new_chat_members`) не спамили чат; антидубль після відправки лишається **15 с**.
- Додано команду **`/here`**: користувач у групі реєструє себе в списку для поділу витрат (актуально при **Group Privacy** і відсутності повного списку в Telegram API). Оновлено підказки в витратах, допомозі та меню.
- Додано **документацію продукту** [`PRODUCT.md`](./PRODUCT.md), **changelog** з вимогою записувати всі зміни, короткий [`README.md`](../README.md) у корені.

Базовий стан продукту на момент заведення документації:

- Груповий біл-спліттер: одна активна поїздка/подія на чат, витрати з поділом, баланси, greedy-розрахунок переказів при завершенні.
- Команди: `/start`, `/menu`, `/new_trip`, `/spent`, `/status`, `/finish_trip`, `/help`; меню з inline-кнопками; згадка @бота.
- `/spent` без аргументів — той самий майстер, що й кнопка витрати.
- Виправлення обліку боргів (частка платника в `Debt`); одноразовий ремонт старих даних при старті для SQLite.
- Допомога з описом продукту та блок підтримки (Mono через env).
- Підтримка PostgreSQL (`asyncpg`) і приклад `DATABASE_URL` у `.env.example`.
- `render.yaml`, `runtime.txt` для деплою на Render (Background Worker).
